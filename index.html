Microsoft Graph API Reference

Ready-to-use queries for Settings Catalog data from your Intune tenant

GET Read operations
List all Settings Catalog policies in tenant
GET https://graph.microsoft.com/beta/deviceManagement/configurationPolicies
Authorization: Bearer {token}
ConsistencyLevel: eventual
Search Settings Catalog definitions by keyword
GET https://graph.microsoft.com/beta/deviceManagement/configurationSettings
  ?$filter=contains(keywords,'{KEYWORD}')
  &$select=id,name,description,keywords,settingDefinitionId,applicability,defaultValue
  &$top=50
Get specific setting definition by ID
GET https://graph.microsoft.com/beta/deviceManagement/configurationSettings/{settingDefinitionId}
Export all settings in a policy (with definitions)
GET https://graph.microsoft.com/beta/deviceManagement/configurationPolicies/{policyId}/settings
  ?$expand=settingDefinitions
POST Create / assign policy
Create Settings Catalog policy with a setting
POST https://graph.microsoft.com/beta/deviceManagement/configurationPolicies

About Settings Explorer

A Helient tool for IT administrators, architects, and endpoint engineers

Purpose

Settings Explorer maps every Windows and macOS policy setting across all configuration methods — GPO/ADMX, Registry, CSP/OMA-URI, and Intune Settings Catalog — in one unified interface. It provides exact Intune setting definition IDs, JSON fragments for Graph API automation, and ready-to-paste registry snippets. Designed specifically for GPO → Intune migrations and cross-method validation.

Data sources
Intune Settings CatalogMicrosoft Graph API (/deviceManagement/configurationSettings)
ADMX / GPOWindows ADMX policy database + Policy Analytics mapping
RegistryWindows registry path reference (HKLM/HKCU)
CSP/OMA-URIWindows MDM configuration service provider documentation
Built-in databaseCurated local dataset of 50+ common settings — no auth required

4Copy the Application (client) ID
